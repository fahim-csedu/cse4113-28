# Lab 6 Peer UI Critique Protocol

Use this protocol to review another team's application **after two implementation sprints**.

## Part A: Instructions

### Objective
Evaluate usability, accessibility, and interface quality with actionable feedback.

### Reviewer Responsibilities
- Explore the app using at least two user roles (if applicable).
- Execute at least 5 key user tasks.
- Identify UI/UX defects, broken flows, and confusing interactions.
- Check responsiveness for desktop and mobile layouts.
- Check baseline accessibility:
  - visible labels
  - keyboard navigation for major flows
  - focus indicators
  - color contrast and readable typography

### Required Output
- Top 5 usability issues (with evidence).
- Top 5 UI improvement suggestions.
- Priority tags: High / Medium / Low.
- Screenshots or short recordings for major findings.
